WHAT · 동작
"반자율"의 의미 — 자료 검색·도구 호출·답변 생성은 자율, 결정·실행은 인간. 발표자료 ⓐ 트랙 5컴포넌트로 구현.
5컴포넌트 동작:
・① Interpreter — 자연어 → JSON 정형 쿼리
・② RAG — 의미 기반 사내 자료 검색
・③ Tool-use — 시뮬·DB·최적화 자동 호출
・④ Explainer — 근거 인용 답변 생성
・⑤ Logger — Gold 데이터셋 누적
4종 도구:
・search_documents — RAG 검색
・query_components — DB 조회
・run_simulation — 1D 시뮬
・suggest_optimization — 최적화
처리량: 시연 기준 1~3초/질문, 비용 $0.05~0.10/회
WHAT NOT · 무엇은 안
최종 결정·실행
(인간 권한)
설계 변경 결정 자체는 인간이 한다
실험 발주·양산 승인 등 책임 작업 안 함
학습되지 않은 영역은 솔직히 "모름" 답변
WHAT NOT · 한계
Phase 1에서 명시적으로 차단되는 작업들. 자율성 L4(자율 에이전트)는 장기 비전이며, 현재 구조에서는 의도적으로 제한.
차단되는 작업:
・최종 설계 결정 (양산·개발 진행 결정)
・실험 발주 (실세계 자원 소모)
・예산 집행 (재무 권한)
・외부 시스템 자동 변경 (PLM·BOM·도면 쓰기 권한)
・학습되지 않은 도메인의 답변 (환각 차단)
왜 제한?:
・안전 책임 — AI 결정의 결과는 인간이 책임지지 않으면 위험
・법적 검토 — 안전 규격·인증 영역은 인간 책임
・회복 가능성 — 잘못된 결정의 회복 비용이 높은 작업은 자율 부적합
장기 비전 (Phase 4+): L3~L4 자율성, 비전문가 확장 — 3~5년+

목적 함수에 대한 후보 자동 탐색 (Bayesian Optimization). 비용 효과 순 정렬.
알고리즘: Bayesian Optimization (BO)
입력: 목적 함수 + 제약
출력: Top-3 후보 + 예상 효과 + 비용
예시: 1순위 — UA 9.2% 증가, 예상 SMER 0.55, +3.7% 비용
✓
규격·가이드 검색 — IEC·EN·사내 가이드
DO · 검색
의미 기반 검색 (RAG). 키워드 안 맞아도 의미가 가까운 자료 찾음.
대상:
・IEC 61121 (건조기 효율 규격)
・EN 60335 (안전 규격)
・사내 설계 가이드 (200+ 문서)
・시험 보고서 (3000+ 건)
특징: 답변에 doc_id 자동 인용 → 사용자가 원문 추적 가능
✓
보고서 자동 생성 — PDF + 3D + 근거
DO · 보고서
분석 결과를 패키지로 — PDF 리포트 + 3D 뷰 + 근거 인용.
포함:
・진단 요약 + 권장 조치
・지표 그래프 (SMER 변화 등)
・3D 컴포넌트 위치 (Three.js)
・근거 자료 인용 + 원문 링크
출력 시간: ~30초
형식: PDF + HTML 임베드
DON'T · 안 한다
Phase 1에서 명시적으로 차단 (5)
✗
최종 설계 결정 — "이 모델로 양산"
DON'T · 결정
설계 변경의 최종 결정은 인간 권한. AI는 후보·근거·예측만 제공.
차단 이유:
・안전 책임 (인증·법적 검토 영역)
・비즈니스 영향 (시장·전략 고려 필요)
・회복 비용 (잘못된 결정의 손실 큼)
대신 AI가 하는 것: 후보 제시, 비교, 근거 정리, 예측 제공
✗
실험·발주·예산 집행
DON'T · 실행
실세계 자원 소모 작업은 차단. 인간 승인 후 수동 실행.
차단 대상:
・실험 발주 (예산·일정 영향)
・예산 집행 (재무 권한)
・벤더 PO 발행
・외부 의뢰 (시험소·인증)
대신 AI가 하는 것: 실험 계획 수립, 비용 추정, 일정 시뮬
✗
모르는 영역 답변 — 환각 차단
DON'T · 환각
학습되지 않은 도메인이나 사내 자료에 없는 내용은 "확인 필요" 답변. 추측 금지.
차단 메커니즘:
・RAG 검색 결과 임계값 (cosine 0.65 미만 → 채택 X)
・"이 자료를 근거로만 답하라" 시스템 프롬프트
・불확실성 표시 (confidence 낮으면 "확인 필요")
오답 줄이는 효과: 환각 비율 30%+ → 5% 미만 (RAG 효과)
✗
비전문가 단독 사용
DON'T · 비전문가
Phase 1에서 비전문가 (기획·임원·현장) 접근 차단. Phase 2~3에서 단계 확장.
차단 이유:
・답변 검증 능력 부재 → 잘못된 답이 의사결정에 영향
・Gold 데이터 라벨링 불가 → Phase 2 sLLM 학습 불가
・안전 책임 (도메인 무지로 인한 위험 인지 불가)
인증 메커니즘: 사내 SSO + 권한 그룹 (R&D 팀만 접근)
✗
외부 시스템 자동 변경
DON'T · 외부 변경
PLM·BOM·도면 시스템 등 외부 시스템에 대한 쓰기 권한 차단. 읽기는 가능.
차단 시스템:
・PLM (제품 수명 주기)
・BOM (자재 명세)
・3D CAD 도면
・ERP (재무·생산)
・PDM (설계 데이터)
읽기 가능 시점: Phase 2 후반 (보안 검토 후)
쓰기 가능 시점: Phase 4+ (장기 비전)
