1 M1~M2 · 2개월 기초
질문 이해하는 기본 시스템 만들기
규격 문서를 검색해서 근거 있는 답변을 주는 최소 기능 시스템(MVP) 구축
🎯이 단계의 목표
"R290 냉매로 바꾸면 안전 규격 뭐 지켜야 해?" 같은 질문에 관련 규격 문서를 찾아서 근거 있는 답변을 주는 시스템
📚먼저 이해해야 할 용어 3개
Claude/GPT API
외부 AI 빌려쓰기
Anthropic의 Claude나 OpenAI의 GPT 같은 이미 만들어진 AI를 돈 내고 쓰는 것. 직접 AI를 만들려면 수억~수십억 원 + 수개월 걸리지만 API는 바로 사용 가능.
비유: 택시 잡아타는 것과 같음. 차 살 필요 없이 돈 내면 목적지까지 태워줌.
RAG
Retrieval-Augmented Generation
AI가 답하기 전에 관련 문서를 먼저 찾아서 참고하게 하는 방식. 회사 내부 문서(규격·실험 데이터)는 AI가 원래 모르니까, 질문 들어오면 관련 문서를 찾아서 "이거 보고 답해"라고 넘겨주는 것.
비유: 시험 볼 때 오픈북 허용하는 것. AI가 "외워서 답하기"가 아니라 "책 찾아보고 답하기"로 바뀜. → 환각(지어내기) 감소.
벡터 DB
의미 기반 검색 DB
문서를 AI가 검색하기 좋은 형태로 저장하는 특수 데이터베이스. 일반 검색은 "단어 일치"로 찾지만, 벡터 DB는 "의미가 비슷한 것"으로 찾음.
예: "냉매 누설" 검색 → 단어가 정확히 없어도 "가스 새는 현상", "refrigerant leak" 같이 의미 유사 문서도 찾아냄.
🔨구체적으로 만드는 것
  1. 01 규격 문서 수집 — IEC 61121, EN 60335 등 PDF 수백 페이지 확보
  2. 02 청킹(Chunking) — 긴 문서를 500~1000자 단위로 잘게 쪼갬 (AI가 한 번에 읽을 수 있게)
  3. 03 임베딩(Embedding) — 잘게 쪼갠 텍스트를 숫자 벡터로 변환해서 벡터 DB에 저장
  4. 04 벡터 DB 선정 — Pinecone / Qdrant / Chroma 중 선택 (사내 보안/비용 고려)
  5. 05 해석기 프롬프트 설계 — 자연어 질문 → JSON 공학 목표로 변환
  6. 06 보안 프록시 — API 호출 전에 회사 기밀 정보(모델명·치수 등) 마스킹
⚙️동작 흐름 예시
질문: "R290 규격 요건?" ↓ [1] 벡터 DB에서 "R290" 관련 문서 Top 5 검색 → IEC 60335-2-11 §7.3 페이지 찾음 ↓ [2] Claude API에 검색 결과 + 질문 전달 ↓ 답변: "R290 충전량 150g 이하... 근거: [IEC 60335-2-11]"
✓ 산출물 (M2 완료 시점)
MVP 시스템 — 규격 관련 질문에 답변 가능. 아직 시뮬레이션이나 최적화는 못 함.
2 M2~M5 · 3개월 에이전트
스스로 일 처리하는 AI 에이전트
AI가 여러 도구(DB·시뮬·최적화)를 스스로 꺼내 쓰면서 답변 생성
🎯이 단계의 목표
"A모델 건조시간 30분 단축 방법은?" 같은 복잡한 질문에 DB 조회 → 시뮬 → 최적화를 알아서 순서대로 실행하고 결과를 자연어로 설명
📚먼저 이해해야 할 용어 3개
에이전트
Agent
단순히 "답만 하는 AI"가 아니라 스스로 판단하고 행동하는 AI. 질문을 이해하고 → 뭘 해야 하는지 계획하고 → 도구를 써서 실행하고 → 결과를 정리하는 일을 스스로 함.
비유: "검색엔진"과 "비서"의 차이. 검색엔진은 링크만 주지만, 비서는 내용을 이해하고 알아서 일을 처리해줌.
Tool-use
도구 사용
AI에게 여러 도구(Tool)를 쥐어주고 필요할 때 꺼내 쓰게 하는 방식. 각 도구는 특정 일을 할 수 있는 프로그램(함수). AI가 질문 맥락 보고 어떤 도구를 어떤 순서로 쓸지 판단.
이 프로젝트의 Tool: ① DB 조회 ② 열유체 시뮬 ③ 최적화 ④ 3D 뷰어 ⑤ 리포트 생성기
LangGraph
에이전트 프레임워크
AI 에이전트를 쉽게 만들게 해주는 도구. 워크플로우를 레고 블록처럼 조립할 수 있음. 분기·반복·에러 복구 같은 복잡한 로직도 그래프로 표현.
비유: PowerPoint의 "도형 그리기"처럼 AI 워크플로우를 그림으로 그려서 만들 수 있음.
🔨구체적으로 만드는 것
  1. 01 오케스트레이터 구현 — AI가 어떤 도구를 어떤 순서로 쓸지 판단하는 두뇌
  2. 02 Tool 5개 등록 — DB 조회 / 물리 시뮬 / 최적화 / 3D 뷰어 / 리포트
  3. 03 LangGraph 상태 그래프 — 분기·반복·에러 복구 로직 설계
  4. 04 설명가(Explainer) — 시뮬 결과(숫자·그래프)를 사람 말로 번역
  5. 05 애매함 감지 + 역질문 — 질문이 부족하면 AI가 먼저 되물음
  6. 06 에러 복구 — 시뮬이 실패해도 다른 방법으로 재시도
⚙️동작 흐름 예시
질문: "A모델 건조시간 30분 줄이려면?" ↓ [AI의 속마음] "먼저 A모델 스펙부터 알아야겠다" → Tool 1: DB 조회 실행 "건조시간은 열유체 문제네" → Tool 2: 열유체 시뮬 실행 "어느 파라미터가 민감한지?" → Tool 3: 최적화 실행 "결과를 정리해야지" → Tool 5: 리포트 생성 ↓ 답변: "팬 RPM 1800→2100, EEV 개도 +5% → 28분 단축 가능"
🎯 M3 마일스톤: MVP 데모 (내부 시연)
✓ 산출물 (M5 완료 시점)

3 M5~M7 · 2개월 멀티도메인
여러 분야 동시에 다루기 + 로그 수집
열유체뿐 아니라 진동·구조·제어까지 통합 + 미래 학습용 데이터 축적 시작
🎯이 단계의 목표

📚먼저 이해해야 할 용어 3개
멀티도메인
Multi-Domain
세탁건조기는 여러 공학 분야가 얽혀있는 제품. 하나 바꾸면 다른 게 따라 움직임. 현실의 설계 질문은 대부분 여러 분야를 동시에 봐야 함.
5개 분야: ① 열유체 (냉매·공기) ② 진동/소음 ③ 구조 ④ 제어 ⑤ 전장
Trade-off
절충/맞교환
하나를 좋게 하면 다른 게 나빠지는 상충 관계. 공학 설계는 거의 항상 trade-off 연속.
예: 팬 RPM ↑ → COP(효율) ↑ 👍 / 소음 ↑ 👎 / 진동 ↑ 👎 → 최적점 찾기가 핵심.
구조화된 로그
Structured Logging
사용자 질문·AI 답변·쓴 도구·결과·피드백을 정해진 형식(JSON)으로 저장하는 것. 나중에 Phase 2에서 자체 AI를 훈련시킬 재료가 됨.
저장 항목: 질문, 답변, 호출한 Tool 순서, 실행 결과, 소요 시간, 사용자 👍/👎
🔨구체적으로 만드는 것
  1. 01 5개 도메인 Tool 통합 — 열유체 외 진동·구조·제어·전장 시뮬 모두 연결
  2. 02 Trade-off 판단 로직 — AI가 여러 목적함수를 비교해서 절충안 찾게 함
  3. 03 로그 수집 파이프라인 — 모든 대화를 JSON으로 자동 저장
  4. 04 피드백 UI — 사용자가 👍/👎 누르면 자동 기록
  5. 05 품질 필터 — 저품질 대화(오류·무의미) 자동 제외
  6. 06 도메인 간 연결 그래프 — "팬 RPM → 소음·COP·진동" 같은 인과 관계 DB화
⚙️동작 흐름 예시
질문: "소음 50dB 이하 + COP 최대화" ↓ [AI가 3개 도메인 동시 고려] → 열유체 시뮬: COP = f(팬 RPM) → 소음 시뮬: dB = g(팬 RPM) → 구조 시뮬: 진동 = h(팬 RPM) ↓ [최적화: 소음 50dB 제약 하에서 COP 최대 지점 탐색] ↓ 답변: "팬 RPM 1950이 최적점 COP 3.2 / 소음 49.8dB / 진동 OK" [동시에 이 대화 전체를 JSON 로그로 저장]

Case A
기존 모델 개선 시나리오
이미 양산 중인 제품을 개선하는 경우. 기준점이 있어서 Case B(새 제품 설계)보다 훨씬 쉬움. 기준 모델에서 얼마나 좋아졌는지만 비교하면 됨.
Case A 질문 예: "A모델 소음 3dB ↓", "R290 전환 시 변경점?", "원가 5000원 절감"
학습 데이터셋
Training Dataset
AI를 훈련시킬 때 쓰는 "문제-정답" 쌍의 모음집. 수천~수만 건 필요. Phase 2에서 자체 sLLM을 만들 때 이걸로 학습시킴.
형식: { 질문: "건조시간 단축 방법?", 정답: "팬 RPM 조정 + EEV 제어..." }
Gold 데이터
검증된 고품질 데이터
전문가가 검증한 신뢰할 수 있는 학습 데이터. 그냥 로그는 틀린 답도 있으니까, 전문가가 리뷰해서 "이건 맞다"고 확인한 것만 Gold로 분류.
비유: 시험 문제집에서 "정답지"가 제대로 맞는지 선생님이 검수한 것.
🔨구체적으로 만드는 것

  2. 02 리포트 자동 생성 — PDF로 최적화 결과 + 근거 + 3D 이미지 포함
  3. 03 로그 정제 — 9개월치 대화 로그를 학습 포맷으로 변환
  4. 04 전문가 리뷰 + Gold 라벨링 — 고품질 답변만 골라서 Gold 데이터로 지정
  5. 05 Synthetic 데이터 증강 — Claude로 유사 질문 자동 생성해서 데이터 양 늘리기
  6. 06 Phase Gate 준비 — M9 전면 리뷰 자료, Phase 2 진입 승인 요청
