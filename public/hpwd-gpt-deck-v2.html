D-2 보충 · AI vs 알고리즘

RAG 안의 AI알고리즘 vs AI · LLM vs 작은 모델

08 / 18
핵심 요약
"의미" → 알고리즘 불가 · AI 필요
단, 작은 전문 AI면 충분 (LLM 불필요)
알고리즘 vs AI
"빨래가 안말라" ≡ "건조 효율 저하"
→ 단어 미일치 · 의미 일치

키워드 · 통계 · 사전 — 의미 처리 불가
"의미" = 명시적 규칙화 불가능 추상 개념
→ 학습된 AI 필요
차이점 — 의미 좌표
알고리즘 · 사람이 명시적 규칙·사전·좌표 직접 입력
임베딩 AI · 수십억 문장 쌍 학습 → 의미 좌표 자동 부여
LLM vs 작은 모델
임베딩·재랭킹 = "의미 거리 측정" 단일 작업
LLM 기능(생성·요약·번역) 불필요

특화 = 작음 · 빠름 · 정확함
LLM 사용 시 — 100~1000× 느림 · 정확도↓
차이점 — 모델 크기·범용성
LLM · 범용 다목적 (생성·요약·번역 등 만능, 수백 GB)
임베딩 모델 · 의미 거리 측정 단일 작업 특화 (수백 MB) — 임베딩은 더 빠르고 정확
우리 시스템 AI 모델 구성
임베딩·재랭킹 (~수백 MB) — ② RAG · 사내 호스팅
외부 LLM API (~수백 GB) — ③ 답변 생성 · 외부 (마스킹)
자체 sLLM (~수십 GB) — Phase 2 외부 LLM 대체 · 사내 호스팅
핵심 — AI ≠ 단일 거대 LLM. 단계별 적정 크기 조합. 대부분 사내 호스팅, 외부 API는 ③에 한정

E-1-4. ⓑ ① DB 설계 · 요소 4

품질 기준 정의좋은 데이터 vs 나쁜 데이터 명시화

17 / 18
ⓑ ① DB 설계 · 5요소요소 4 강조
한 줄 정리
자동 검증 가능한 명시적 룰
"좋은 데이터" 판단 기준 → 코드화 → ⑥ 검증·품질 트랙의 입력
4가지 차원
완결성 (Completeness) — 필수 필드 채워짐? 미충족 시 reject
일관성 (Consistency) — 같은 의미 = 같은 형식? cc · ㎤ 통일 강제
정확성 (Accuracy) — 값이 사실과 일치? 음수 압력 자동 검출
최신성 (Timeliness) — 데이터 수명? 5년 이상 → "오래됨" 라벨
구체 룰 예시
· 실험 온도 −40°C ~ 60°C 외 → reject
· 압축기 RPM 0 ~ 7000 외 → flag
· 시계열 샘플링 변동 30% 초과 → warning
· 시험 보고서 12개월 미갱신 → "오래됨"
필요성
AI 답변 품질 = 데이터 품질
잘못된 데이터 학습 → AI 잘못된 답변

자동 검증 — 룰 코드화 시 신규 데이터 자동 체크
부서 간 데이터 거래 기준 — "이 형식으로 주세요" 근거
